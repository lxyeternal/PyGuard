778	Call signature::
779	
780	  barbs([X, Y], U, V, [C], /, **kwargs)
781	
782	Where *X*, *Y* define the barb locations, *U*, *V* define the barb
783	directions, and *C* optionally sets the color.
784	
785	The arguments *X*, *Y*, *U*, *V*, *C* are positional-only and may be
786	1D or 2D. *U*, *V*, *C* may be masked arrays, but masked *X*, *Y*
787	are not supported at present.
788	
789	Barbs are traditionally used in meteorology as a way to plot the speed
790	and direction of wind observations, but can technically be used to
791	plot any two dimensional vector quantity.  As opposed to arrows, which
792	give vector magnitude by the length of the arrow, the barbs give more
793	quantitative information about the vector magnitude by putting slanted
794	lines or a triangle for various increments in magnitude, as show
795	schematically below::
796	
797	  :                   /\    \
798	  :                  /  \    \
799	  :                 /    \    \    \
800	  :                /      \    \    \
801	  :               ------------------------------
802	
803	The largest increment is given by a triangle (or "flag"). After those
804	come full lines (barbs). The smallest increment is a half line.  There
805	is only, of course, ever at most 1 half line.  If the magnitude is
806	small and only needs a single half-line and no full lines or
807	triangles, the half-line is offset from the end of the barb so that it
808	can be easily distinguished from barbs with a single full line.  The
809	magnitude for the barb shown above would nominally be 65, using the
810	standard increments of 50, 10, and 5.
811	
812	See also https://en.wikipedia.org/wiki/Wind_barb.
813	
814	Parameters
815	----------
816	X, Y : 1D or 2D array-like, optional
817	    The x and y coordinates of the barb locations. See *pivot* for how the
818	    barbs are drawn to the x, y positions.
819	
820	    If not given, they will be generated as a uniform integer meshgrid based
821	    on the dimensions of *U* and *V*.
822	
823	    If *X* and *Y* are 1D but *U*, *V* are 2D, *X*, *Y* are expanded to 2D
824	    using ``X, Y = np.meshgrid(X, Y)``. In this case ``len(X)`` and ``len(Y)``
825	    must match the column and row dimensions of *U* and *V*.
826	
827	U, V : 1D or 2D array-like
828	    The x and y components of the barb shaft.
829	
830	C : 1D or 2D array-like, optional
831	    Numeric data that defines the barb colors by colormapping via *norm* and
832	    *cmap*.
833	
834	    This does not support explicit colors. If you want to set colors directly,
835	    use *barbcolor* instead.
836	
837	length : float, default: 7
838	    Length of the barb in points; the other parts of the barb
839	    are scaled against this.
840	
841	pivot : {'tip', 'middle'} or float, default: 'tip'
842	    The part of the arrow that is anchored to the *X*, *Y* grid. The barb
843	    rotates about this point. This can also be a number, which shifts the
844	    start of the barb that many points away from grid point.
845	
846	barbcolor : :mpltype:`color` or color sequence
847	    The color of all parts of the barb except for the flags.  This parameter
848	    is analogous to the *edgecolor* parameter for polygons, which can be used
849	    instead. However this parameter will override facecolor.
850	
851	flagcolor : :mpltype:`color` or color sequence
852	    The color of any flags on the barb.  This parameter is analogous to the
853	    *facecolor* parameter for polygons, which can be used instead. However,
854	    this parameter will override facecolor.  If this is not set (and *C* has
855	    not either) then *flagcolor* will be set to match *barbcolor* so that the
856	    barb has a uniform color. If *C* has been set, *flagcolor* has no effect.
857	
858	sizes : dict, optional
859	    A dictionary of coefficients specifying the ratio of a given
860	    feature to the length of the barb. Only those values one wishes to
861	    override need to be included.  These features include:
862	
863	    - 'spacing' - space between features (flags, full/half barbs)
864	    - 'height' - height (distance from shaft to top) of a flag or full barb
865	    - 'width' - width of a flag, twice the width of a full barb
866	    - 'emptybarb' - radius of the circle used for low magnitudes
867	
868	fill_empty : bool, default: False
869	    Whether the empty barbs (circles) that are drawn should be filled with
870	    the flag color.  If they are not filled, the center is transparent.
871	
872	rounding : bool, default: True
873	    Whether the vector magnitude should be rounded when allocating barb
874	    components.  If True, the magnitude is rounded to the nearest multiple
875	    of the half-barb increment.  If False, the magnitude is simply truncated
876	    to the next lowest multiple.
877	
878	barb_increments : dict, optional
879	    A dictionary of increments specifying values to associate with
880	    different parts of the barb. Only those values one wishes to
881	    override need to be included.
882	
883	    - 'half' - half barbs (Default is 5)
884	    - 'full' - full barbs (Default is 10)
885	    - 'flag' - flags (default is 50)
886	
887	flip_barb : bool or array-like of bool, default: False
888	    Whether the lines and flags should point opposite to normal.
889	    Normal behavior is for the barbs and lines to point right (comes from wind
890	    barbs having these features point towards low pressure in the Northern
891	    Hemisphere).
892	
893	    A single value is applied to all barbs. Individual barbs can be flipped by
894	    passing a bool array of the same size as *U* and *V*.
895	
896	Returns
897	-------
898	barbs : `~matplotlib.quiver.Barbs`
899	
900	Other Parameters
901	----------------
902	data : indexable object, optional
903	    DATA_PARAMETER_PLACEHOLDER
904	
905	**kwargs
906	    The barbs can further be customized using `.PolyCollection` keyword
907	    arguments:
908	
909	    %(PolyCollection:kwdoc)s
910	""" % _docstring.interpd.params
