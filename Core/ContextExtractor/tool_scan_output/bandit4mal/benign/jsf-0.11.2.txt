67	Use **jsf** along with fake data generators to provide consistent and meaningful fake data for your system.
68	
69	## Main Features
70	
71	- Provides out of the box data generation from any JSON schema üì¶
72	- Extendable custom data providers using any lambda functions üîó
73	- Multi level state for dependant data (eg multiple objects sharing value, such as children with same surname) ü§ì
74	- Inbuilt validation of fake JSON produced ‚úÖ
75	- In memory conversion from JSON Schema to Pydantic Models with generated examples ü§Ø
76	- Seamless integration with [FastAPI](https://fastapi.tiangolo.com/) üöÄ

90	## Usage
91	
92	### Basic üòä
93	
94	```python
95	from jsf import JSF
96	
97	faker = JSF(
98	    {
99	        "type": "object",
100	        "properties": {
101	            "name": {"type": "string", "$provider": "faker.name"},
102	            "email": {"type": "string", "$provider": "faker.email"},
103	        },
104	        "required": ["name", "email"],
105	    }
106	)
107	
108	fake_json = faker.generate()
109	```
110	
111	Results in ...
112	
113	```python
114	{
115	    'name': 'Jesse Phillips', 
116	    'email': 'xroberson@hotmail.com'
117	}
118	```
