8	About Dill
9	==========
10	
11	``dill`` extends python's ``pickle`` module for serializing and de-serializing
12	python objects to the majority of the built-in python types. Serialization
13	is the process of converting an object to a byte stream, and the inverse
14	of which is converting a byte stream back to on python object hierarchy.
15	
16	``dill`` provides the user the same interface as the ``pickle`` module, and
17	also includes some additional features. In addition to pickling python
18	objects, ``dill`` provides the ability to save the state of an interpreter
19	session in a single command.  Hence, it would be feasable to save a
20	interpreter session, close the interpreter, ship the pickled file to
21	another computer, open a new interpreter, unpickle the session and
22	thus continue from the 'saved' state of the original interpreter
23	session.
24	
25	``dill`` can be used to store python objects to a file, but the primary
26	usage is to send python objects across the network as a byte stream.
27	``dill`` is quite flexible, and allows arbitrary user defined classes
28	and functions to be serialized.  Thus ``dill`` is not intended to be
29	secure against erroneously or maliciously constructed data. It is
30	left to the user to decide whether the data they unpickle is from
31	a trustworthy source.
32	
33	``dill`` is part of ``pathos``, a python framework for heterogeneous computing.
34	``dill`` is in active development, so any user feedback, bug reports, comments,
35	or suggestions are highly appreciated.  A list of known issues is maintained
36	at http://trac.mystic.cacr.caltech.edu/project/pathos/query.html, with a public
37	ticket list at https://github.com/uqfoundation/dill/issues.
38

40	Major Features
41	==============
42	
43	``dill`` can pickle the following standard types:
44	
45	    - none, type, bool, int, long, float, complex, str, unicode,
46	    - tuple, list, dict, file, buffer, builtin,
47	    - both old and new style classes,
48	    - instances of old and new style classes,
49	    - set, frozenset, array, functions, exceptions
50	
51	``dill`` can also pickle more 'exotic' standard types:
52

55	    - dictproxy, methoddescriptor, getsetdescriptor, memberdescriptor,

60
